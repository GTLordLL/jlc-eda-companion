# Step 1 交叉对比报告：手册要求 vs ERC 实际

> 样本工程：51单片机最小系统板.eprj2
> 核心芯片：STC89C52RC-40I-LQFP44 (C8707)
> 手册版本：STC89/90 系列技术手册 (2025/7/15)
> ERC 引擎：phase2_erc_check.py (7项通用规则)

## 一、7 项 ERC 检查逐项交叉对比

### 1. IC VCC 去耦电容 (`decoupling_cap`)

| 维度 | 内容 |
|------|------|
| **手册要求** | §3.4.9 LQFP44 最小系统图：MCU-VCC(Pin44) 接 **0.1µF (100nF)** 到 GND；另手册 §3.4 电源电路有 10µF+100nF 组合 |
| **ERC 发现** | ❌ U1 Pin2 (VCC+5V) 未检测到 100nF；❌ U2 Pin44 (VCC+5V) 未检测到 100nF |
| **实际原理图** | C4 (值未标注) 可能为 100nF，但 C4 可能不在 VCC+5V 网络上（ERC 未检出）；C5 (值未标注) 也可能为去耦电容 |
| **判定** | **✅ 确认真实问题** — C4/C5 未在原理图中标注值，ERC 无法识别；即便存在，手册要求 100nF **就近** (<5mm) 放置于 Pin44 旁，需人工确认 Layout |
| **严重度** | ❌ 错误 — 缺少去耦可导致 MCU 工作不稳定 |

### 2. EN/复位引脚上下拉 (`enable_pin`)

| 维度 | 内容 |
|------|------|
| **手册要求** | §5.1.2 复位：RST (Pin?) 需 10KΩ 上拉 VCC + 10µF 到 GND。§3.6 管脚说明：**PSEN/P4.4 是"外部 ROM 选通"**，非 EN/复位引脚 |
| **ERC 发现** | ⚠️ U2 Pin25 (PSEN#) — "EN/复位引脚未检测到上下拉电阻"；⚠️ H6 Pin9 (PSEN#) — 同上 |
| **实际原理图** | PSEN# 网络仅连接 U2.Pin25 和 H6.Pin9 (2 pins)，无上拉/下拉电阻 |
| **判定** | **❌ 误报 (False Positive)** — ERC 规则将"PSEN"匹配为 EN 引脚模式 (`pin_pattern: "EN"`)，但手册明确定义 PSEN 为 Program Store Enable（外部 ROM 选通），非 Enable/复位引脚。STC89C52RC 无外部 ROM 时 PSEN 可浮空 |
| **严重度** | 误报 — 从 ERC 报告中排除 |

### 3. I2C 上拉电阻 (`i2c_pullup`)

| 维度 | 内容 |
|------|------|
| **ERC 发现** | ✅ 通过 — 总线上未检测到 I2C 设备 |
| **判定** | **✅ 正确通过** — STC89C52RC 无硬件 I2C 模块（仅有 UART），样本工程未使用 I2C 外设 |

### 4. 电源反馈分压 (`feedback_divider`)

| 维度 | 内容 |
|------|------|
| **ERC 发现** | ✅ 通过 — AMS1117-3.3 是固定输出 LDO，无反馈分压电阻 |
| **判定** | **✅ 正确通过** — AMS1117-3.3 内部固定 3.3V 输出，不需要外部反馈分压 |

### 5. 晶振负载电容 (`crystal_load_cap`)

| 维度 | 内容 |
|------|------|
| **手册要求** | §25.6 外部晶振输入电路：晶振两脚各接电容到 GND，典型值 **22-33pF**。手册未指定 12MHz 晶振的精确 CL |
| **ERC 发现** | 💡 X1 C6,C7=47pF → CL_eff = (47×47)/(47+47) + 3pF = **26.5pF**。需对照晶振手册 CL 验证 |
| **实际原理图** | C6=C7=47pF，X1=12MHz (C295072, HC-49S 直插) |
| **compute_passive 计算** | 若目标 CL=20pF → C1=C2=34pF → 标准值 **33pF** (−2.94%) |
| **判定** | **⚠️ 需确认** — 需查阅 C295072 晶振规格书的 CL 值。若 CL=20pF，则 47pF 偏大 (+32.5%)，建议改 33pF；若 CL=30pF，则 47pF 接近（CLeff=26.5pF，偏差 −11.7%） |
| **严重度** | ⚠️ 警告 — 不影响起振但可能导致频率偏移 |

### 6. 悬空引脚 (`floating_pin`)

| 维度 | 内容 |
|------|------|
| **手册要求** | §3.6：未用 I/O 引脚可浮空（8051 内置弱上拉）；P0 口例外需外接上拉 |
| **ERC 发现** | 💡 LED1 Pin1 (Net_6, 1pin 悬空)；💡 U2 Pin14 (RD#, 1pin 悬空)；💡 U2 Pin18 (P2.1, 1pin 悬空) |
| **实际原理图** | LED1 单脚悬空可能是未完成的 LED 电路；RD# 是外部 RAM 读信号（无外部 RAM 时可浮空）；P2.1 是 I/O（可浮空） |
| **判定** | **💡 建议排除** — LED1 是"半完成电路"需人工处理；RD# 无外部 RAM 时可浮空；P2.1 内置上拉可浮空。均为低风险 |

### 7. 电源-地短路 (`power_ground_short`)

| 维度 | 内容 |
|------|------|
| **ERC 发现** | ✅ 通过 — 未检测到电源与地短路 |
| **判定** | **✅ 正确通过** |

## 三、ERC 引擎改进建议

### 3.1 误报原因

`phase2_erc_check.py` 的 `enable_pin` 检查使用 `pin_pattern: "EN"` 匹配引脚名，
将 **PSEN** 误匹配。应排除已知的非 EN 引脚：

```
PSEN, PSEN#, PSEN~  →  8051 外部程序存储使能，非 Enable 引脚
```

### 3.2 通用规则的局限

当前通用规则无法区分：
- **100nF 去耦电容在电路中但 ERC 找不到** — 因为 C4/C5 的 `value` 字段为空
- **晶振负载电容是否匹配** — 需知道晶振的实际 CL 值

这两个短板正是 **Step 2 (Design Spec) + Step 3 (Spec驱动ERC)** 要解决的。
