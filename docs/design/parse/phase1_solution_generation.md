# 阶段一：解决方案生成

> 合并原"阶段一：需求定义与架构设计"和"阶段二：芯片选型与方案验证"
> 基于讨论：两种设计路径最终都指向芯片数据手册

---

## 一、设计理念

### 1.1 目标用户画像

本 Skill 面向**非电子专业的个人开发者**。这类用户的基本设计方法是：

```
选定芯片 → 读数据手册 → 抄参考电路 → 欧姆定律/基尔霍夫定律算阻容值 → 画原理图
```

他们不会设计复杂的模拟电路，但能够理解手册中的典型应用电路，并按照手册的推荐参数搭建外围电路。对他们来说，**芯片数据手册是最重要的参考物，没有之一**。

### 1.2 两种设计路径

电子设计本质上只有两种起点，但最终殊途同归：

```
路径A（有轮子）：找到成熟方案 → 方案用了什么芯片 → 看芯片手册的参考电路
路径B（自己造）：选定一颗芯片 → 看芯片手册的参考电路 → 照做 + 算参数
                                                    ↓
                                   最终都落在「解读和应用芯片手册」
```

| | 路径A：参考方案 | 路径B：芯片选型 |
|---|---|---|
| 适用场景 | 需求明确，市面有类似产品 | 需求特殊，或想用特定芯片 |
| 入口 | LCSC 搜索、开源硬件社区 | LCSC 品类筛选 |
| 核心动作 | 定位方案中的关键芯片 | 根据参数选定芯片 |
| 收敛点 | ↓ | ↓ |
| **共同终点** | **下载芯片数据手册 → 提取参考电路 → 生成外围 BOM** | 同左 |

### 1.3 数据手册是唯一权威

```
信息源可靠性排序：
1. 芯片厂商数据手册 (PDF) ← 唯一权威，以此为准则
2. LCSC 产品页 (HTML)     ← 辅助参考，可能有错误或遗漏
3. Claude 硬件知识         ← 兜底推理，可能不准确
```

**LCSC 产品页的信息可能出错。** 必须以芯片手册中的信息为准。LCSC 的作用是：
- 搜索和筛选芯片（品类、参数、库存、价格）
- 提供 Datasheet PDF 的下载链接
- 提供封装、制造商等元数据

## 二、两种设计路径的详细流程

### 路径A：参考方案搜索

```
用户需求（如："做一个智能家居温控器"）
  ↓
① Claude 引导需求澄清
  - 供电方式？（电池/USB/DC适配器，电压范围）
  - 需要哪些外设接口？（I2C/SPI/UART/WiFi...）
  - 尺寸/成本约束？
  ↓
② 输出系统架构草稿
  - 电源树（24V→5V→3.3V）
  - 信号总线分配（I2C-1: MCU→传感器, UART: MCU→WiFi）
  - 关键芯片类型清单（MCU、温湿度传感器、继电器驱动、电源芯片）
  ↓
③ 搜索现有方案
  - LCSC 搜索：按品类+参数筛选在售器件
  - 开源硬件社区：GitHub/立创开源广场搜索类似项目
  - 方案对比：列出 2-3 个可行方案及其核心芯片
  ↓
④ 选定方案 → 进入「手册解读」环节
```

### 路径B：芯片选型 + 外围电路设计

```
用户需求（如："用STM32F103C8T6做一个CAN总线数据采集器"）
  ↓
① 确认核心芯片已在手
  ↓
② 获取数据手册
  - 优先：LCSC 产品页的 Datasheet 链接 → 下载 PDF
  - 备用：制造商官网搜索
  ↓
③ Docling MCP 解析 PDF → 提取关键章节
  - Pinout & Pin Description（引脚定义）
  - Typical Application Circuit（典型应用电路）
  - Reference Design（参考设计，含阻容推荐值）
  - Electrical Characteristics（电气特性，电压/电流范围）
  - Layout Guidelines（PCB 布局指导）
  ↓
④ Claude 从 Markdown 中提取结构化信息
  - 电源引脚：VCC/VDD 数量、电压范围、推荐去耦方案
  - 关键外围：晶振频率+负载电容、复位电路、Boot 配置
  - 通讯接口：I2C/SPI/UART 对应的引脚映射
  - 参考电路标注的外围器件值（R1=10K, C1=100nF ...）
  ↓
⑤ 生成外围 BOM 清单
  - 对每个外围器件，在 LCSC 找到有库存的具体型号
  - 计算需调整的参数（反馈电阻分压、滤波截止频率等）
  - 输出：设计文档 + BOM 表
```

## 三、技术实现

### 3.1 PDF 解析方案：Docling（IBM 开源）

**不自己写 PDF 解析。** 直接使用 IBM Docling 的 MCP Server：

| 维度 | 选择 | 理由 |
|---|---|---|
| 项目 | `docling-project/docling` | IBM 开源，专为 LLM 生态设计 |
| 集成方式 | **MCP Server**（原生支持） | Claude Code 可直接调用，无需封装 |
| 输出格式 | Markdown | LLM 可直接理解的结构化文本 |
| 核心能力 | PDF/DOCX/PPTX/XLSX→Markdown | 保留表格、标题层级、列表结构 |

**Docling MCP 在本阶段的作用：**

```
用户提供 Datasheet PDF 路径
  → Claude 调用 Docling MCP: "解析这个 PDF，提取第X章典型应用电路"
  → Docling 返回 Markdown 文本（含表格、列表）
  → Claude 从 Markdown 中提取：引脚定义、参考电路连接关系、推荐阻容值
```

### 3.2 电路图识别：不做

| 方案 | 可行性 | 理由 |
|---|---|---|
| LLM 视觉识别电路图 | ❌ | LLM 对电路图理解能力很弱 |
| YOLO/CNN 训练电路识别 | ❌ | 工程量大，不在本 Skill 范围内 |
| **读取手册文字描述** | ✅ | 手册文字已包含参考电路的全部连接关系和参数 |

数据手册的典型应用章节结构：
```
[电路图图片]         ← 给人类看的，跳过
[文字描述段落]       ← LLM 读取
  "A 10µF ceramic capacitor should be connected between
   VIN (pin 2) and GND (pin 1). A 100nF bypass capacitor
   must be placed as close as possible to the VOUT pin.
   The feedback resistor divider R1 and R2 sets the output
   voltage according to VOUT = 0.8 × (1 + R1/R2).
   Recommended values: R2 = 10kΩ, R1 = 52.3kΩ for 5V output."
```

文字描述已经足够涵盖：引脚连接关系、推荐元件值、计算公式、Layout 注意事项。

### 3.3 工具清单

| 工具 | 用途 | 实现方式 |
|---|---|---|
| Docling MCP Server | 解析 Datasheet PDF → Markdown | 已有开源方案，挂载即可 |
| `search_lcsc.py` | LCSC 品类搜索 + 库存查询 | Python, 调用 LCSC API |
| `parse_jlc_project.py` | 解析嘉立创工程文件（三种格式） | Python, 本 Skill 核心工具 |
| `compute_passive.py` | 欧姆定律/分压/滤波计算器 | Python, 纯数学计算 |

### 3.4 「方案阶段」的输出物

完成本阶段后，用户应获得：

```markdown
## 设计方案：智能家居温控器

### 系统架构
[电源树 + 信号总线图]

### 核心芯片清单
| 位号 | 芯片 | LCSC 编号 | 封装 | 用途 | 库存 |
|------|------|-----------|------|------|------|
| U1 | STM32F103C8T6 | C8734 | LQFP48 | MCU | ✅ 5000+ |
| U2 | AMS1117-3.3 | C6186 | SOT-223 | 3.3V LDO | ✅ 10000+ |
| U3 | SHT30-DIS-B | C99235 | DFN-8 | 温湿度传感器 | ✅ 3000+ |

### 外围电路 BOM（基于手册参考电路）
| 位号 | 器件 | 值 | LCSC 编号 | 依据 |
|------|------|-----|-----------|------|
| C1,C2 | 去耦电容 | 100nF | C15849 | 手册 §5.2: VDD 各引脚就近接 100nF |
| C3 | LDO 输出电容 | 10µF | C15849 | 手册 §6.1: COUT ≥ 10µF 钽电容 |
| R1 | NRST 上拉 | 10KΩ | C17414 | 手册 §4.3: NRST 内部弱上拉，外部建议 10K |
| Y1 | 晶振 | 8MHz | C12674 | 手册 §4.1: HSE 4-16MHz，推荐 8MHz |
| C4,C5 | 晶振负载电容 | 20pF | C1540 | 手册 §4.1: CL=20pF 匹配晶振 |

### 关键计算公式
- VOUT = 0.8 × (1 + R1/R2) = 0.8 × (1 + 52.3K/10K) = 5.0V ✅
```

## 四、与后续阶段的衔接

方案阶段输出的 BOM 清单和设计参数，将直接用于：

```
方案阶段（本阶段）
  ├── 核心芯片清单 → 嘉立创 EDA 中放置元件
  ├── 外围 BOM     → 阶段二 ERC：检查原理图是否与手册一致
  ├── 计算公式     → 阶段二 ERC：验证反馈电阻、滤波参数
  └── Layout 要求   → 阶段三 PCB：检查布局是否符合手册指南
```

## 五、关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| PDF 解析 | Docling MCP | IBM 开源，原生 MCP Server，无需自建 |
| 电路图识别 | 不做 | LLM 视觉不可靠，文字描述已足够 |
| 信息权威源 | 芯片数据手册 | LCSC 页面可能出错，手册是唯一权威 |
| 目标用户 | 非专业开发者 | 会抄参考电路 + 会欧姆定律即可完成设计 |
| 阶段合并 | 一+二→方案阶段 | 两种路径殊途同归，核心都是手册解读 |
